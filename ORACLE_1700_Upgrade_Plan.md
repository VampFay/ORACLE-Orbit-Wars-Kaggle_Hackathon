# ORACLE 1700+ ELO Upgrade Plan

To push the ORACLE bot from ~1550 ELO to 1700+ and beat the leaderboard toppers, we are breaking down the improvements into four modular components. We will implement them step-by-step.

## Phase 1: Precision Defense (Just-in-Time Reinforcements)
**Concept**: Currently, the heuristic reacts to threats by immediately sending reinforcements. This pulls ships away from your production planets too early, starving your economy.
**Implementation**: 
1. Create `detailed_threat()` to return not just the threat size, but the *exact ETA* of the incoming fleet.
2. Calculate the transit time from your donor planet to the threatened planet.
3. Only launch the reinforcement fleet when `current_step >= ETA - travel_time - 1`. 

## Phase 2: Time-Synchronized Coordinated Attacks
**Concept**: Top bots attack single planets from multiple sources so that all fleets arrive on the *exact same tick*, bypassing regeneration.
**Implementation**:
1. Group attack candidates by target planet instead of executing them greedily.
2. Determine the maximum arrival time among the launching planets.
3. Delay the launch of closer planets so their arrival time matches the furthest planet.

## Phase 3: MCTS Action Abstraction
**Concept**: MCTS currently searches single-fleet launches (1 move/turn). We will change the MCTS action space so each node represents an entire heuristic strategy (e.g., all moves generated by Phase 1+2+4 of the heuristic).
**Implementation**:
1. Update `generate_candidate_actions()` to return a list of full-turn move sets instead of single actions.
2. Re-enable MCTS as the primary decision engine for turns 30-450, evaluating these abstracted heuristic moves.

## Phase 4: Parameter Tuning (CMA-ES Script)
**Concept**: Hardcoded thresholds (`counter_threshold=20`, `avail < 8`, `horizon=8`) are good guesses, but not mathematically optimal. 
**Implementation**:
1. Create `tune_oracle.py`, a standalone script that runs thousands of headless matches in parallel.
2. Expose the magic numbers in `main.py` to be readable from the `config` object.
3. Use a basic evolution strategy to iteratively mutate and discover the perfect values.
